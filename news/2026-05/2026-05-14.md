### 2. Hugging Face、LLM推論の非同期連続バッチ処理技術を公開

| 項目 | 内容 |
|------|------|
| ソース | [Hugging Face](https://huggingface.co/blog/continuous_async) |
| 公開日 | 2026-05-14 |
| 重要度 | ⭐⭐⭐ |

**概要**: Hugging FaceがLLM推論におけるGPU利用率を大幅に向上させる非同期連続バッチ処理技術を詳細に解説。CPUとGPUのワークロードを分離し、両方を並列実行することで、GPUのアイドル時間を削減する技術を公開。

**詳細ポイント**:
- 同期バッチ処理ではGPUが24%の時間アイドル状態だったのが、非同期化により99.4%の時間アクティブに改善
- CUDAストリームとイベントを使用してCPUとGPUのワークロードを調整し、データ競合を防止
- 8Bモデルを使用した実験で、8Kトークン生成の時間を300.6秒から234.5秒に短縮（22%の速度向上）

**なぜ重要か**: この技術は、新しいモデルやカーネルの変更なしに、既存のハードウェアでの推論スループットを大幅に改善する。特に長文生成やエージェントアプリケーションにおいて、計算リソースの効率的な利用を可能にする。

**関連キーワード**: 推論最適化, GPU利用率, CUDAストリーム, 連続バッチ処理, transformers

### 2. Claude Codeの'/goals'、作業を行うエージェントと完了を判断するエージェントを分離

| 項目 | 内容 |
|------|------|
| ソース | [VentureBeat](https://venturebeat.com/orchestration/claude-codes-goals-separates-the-agent-that-works-from-the-one-that-decides-its-done/) |
| 公開日 | 2026-05-14 |
| 重要度 | ⭐⭐⭐ |

**概要**: AnthropicのClaude Codeが新しい'/goals'コマンドを導入し、タスク実行とタスク評価を正式に分離。このアプローチにより、AIエージェントが「実際に終わった」と判断する前に作業を終了するという一般的な問題を解決する。

**詳細ポイント**:
- ユーザーが目標を定義すると、Claudeはターンごとに動作を続け、各ステップの後に評価モデル（デフォルトはHaiku）が目標が達成されたかどうかを確認
- 最も成功する条件は通常、測定可能な最終状態（テスト結果、ビルド終了コード、ファイル数など）、Claudeがそれを証明する方法（「npm test exits 0」など）、変更してはならない制約を明記すること
- このアプローチにより、サードパーティの観測性プラットフォームの必要性がなくなり、エージェントがすでに達成したこととまだ行う必要があることを混同するのを防ぐ

**なぜ重要か**: この機能は、AIエージェントの信頼性と監査可能性を向上させ、企業がパイロットから本番環境への移行を加速させることを可能にする。これは状態維持、長期実行、自己学習型エージェントの可能性が高まる中で、エージェントやオーケストレーションをより監査可能で観察可能なシステムへと押し上げる大きなトレンドの一部である。

**関連キーワード**: Claude Code, エージェント評価, 目標指向AI, タスク完了検証, Anthropic
